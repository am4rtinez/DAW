Operacions sobre usuaris

En aquest apartat es defineixen les diferents operacions que es poden realitzar sobre usuaris

Endpoints usuaris
GET /gimnas/usuari
POST /gimnas/usuari
GET /gimnas/usuari/{id_user}
PUT /gimnas/usuari/{id_user}
DELETE /gimnas/usuari/{id_user}

Llista usuaris

Retorna una llista amb tots els camps de la llista usuaris excepte el password.

GET /gimnas/usuari

Afegir usuari

Afegeix un usuari (fent la conversió a hash del password).

En aquesta versio de l'API es pot repetir l'username ja que no es un camp unic. (Futura modificacio a la BD).

El JSON ha de tenir el seguent format:

POST /gimnas/usuari
  • username - Username de l'usuari.
  • nom - Nom de l'usuari.
  • llinatges llinatges de l'usuari.
  • email - Email de contacte.
  • telefon - Telefon de contacte.
  • password - El sistema s'encarregara d'encriptar-la abans d'enmagatzemar-la.

Quan es faci el instert retorna l'id del usuari (autogenerat amb l'insert).

Llista info usuari

Retorna els detalls de l'usuari, excepte el password, amb l'id de la url.

GET /gimnas/usuari/{id_user}

Modifica info usuari

Modifica els camps d'un usuari. Si es modifica el password el sistema fa el hash.

El JSON ha de tenir el seguent format:

PUT /gimnas/usuari/{id_user}
  • username - Username de l'usuari.
  • nom - Nom de l'usuari.
  • llinatges llinatges de l'usuari.
  • email - Email de contacte.
  • telefon - Telefon de contacte.
  • password - El sistema s'encarregara d'encriptar-la abans d'enmagatzemar-la.

No es necessari que contengui tots els camps. Nomes el camps que es vulguin modificar.

Elimina usuari

Elimina un usuari a partir de la id en el parametre de la url.

Previament a la eliminacio de l'usuari eliminara les reserves d'aquest.

Si no hi ha usuari amb aquest id no fa res.

DELETE /gimnas/usuari/{id_user}

Operacions sobre reserves

En aquest apartat es defineixen les diferents operacions que es poden realitzar sobre reserves

Endpoints reserves
GET /gimnas/reserves
GET /gimnas/reserves/setmana/{data}
GET /gimnas/reserves/usuari/{id_user}
POST /gimnas/reserves/usuari/{id_user}
DELETE /gimnas/reserves/usuari/{id_user}

Llista reserves

Retorna totes les reserves de la setmana actual (a partir de datetime.now o today de python). Ha de retornar el nom de la pista i el username de l'usuari.

GET /gimnas/reserves

Llista reserves data

Retorna totes les reserves de la setmana indicada a "data" (en format YYYY-mm-dd ). Ha de retornar el nom de la pista i el username de l'usuari.

GET /gimnas/reserves/setmana/{data}

Llista reserves usuari

Retorna totes les reserves del l'usuari amb id=id_user, indicant el nom de la pista.

GET /gimnas/reserves/{id_user}

Afegir reserves usuari

Afegeix una reserva de l'usuari "id_usuari", els detalls venen a un JSON.

POST /gimnas/reserves/{id_user}
  • data - Data de la reserva amb format "YYYY-mm-dd HH:MM:ss".
  • idpista - Identificador de la pista (1 - Coberta / 2 - Exterior).

Elimina reserves usuari

Elimina una reserva de l'usuari "id_usuari", els detalls venen a un JSON.

DELETE /gimnas/reserves/{id_user}
  • data - Data de la reserva amb format "YYYY-mm-dd HH:MM:ss".
  • idpista - Identificador de la pista (1 - Coberta / 2 - Exterior).
