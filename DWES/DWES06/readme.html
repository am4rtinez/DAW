Tasca DWES06 - Gimnàs API REST

API Reference

API REST per gestionar el Gimnas DWES.

BASE URL
http://localhost:5000

Operacions sobre usuaris

Endpoints:
GET /gimnas/usuari
POST /gimnas/usuari

GET

Retorna una llista amb tots els camps de la llista usuaris excepte el password.

POST

Afegeix un usuari (fent la conversió a hash del password).

El JSON ha de tenir el siguent format:

  • username - Username de l'usuari.
  • nom - Nom de l'usuari.
  • llinatges llinatges de l'usuari.
  • email - Email de contacte.
  • telefon - Telefon de contacte.
  • password - El sistema s'encarregara d'encriptar-la abans d'enmagatzemar-la.

Quan es faci el instert retorna l'id del usuari (autogenerat amb l'insert).

Endpoints:
GET /gimnas/usuari/id
PUT /gimnas/usuari/id
DELETE /gimnas/usuari/id
GET: retorna els detalls de l'usuari excepte el password
PUT: Modifica els camps d'un usuari. Si és el password s'ha de fer el hash
DELETE: Elimina un usuari

Operacions sobre reserves

Endpoint:
GET /gimnas/reserves
GET: Retorna totes les reserves de la setmana actual (a partir de datetime.now o today de python). Ha de retornar el nom de la pista i el username de l'usuari.
Endpoint:
GET /gimnas/reserves/setmana/data
GET: Retorna totes les reserves de la setmana indicada a "data" (en format YYYY-mm-dd ). Ha de retornar el nom de la pista i el username de l'usuari
Endpoints:
GET /gimnas/reserves/usuari/id_usuari
POST /gimnas/reserves/usuari/id_usuari
DELETE /gimnas/reserves/usuari/id_usuari
GET: Retorna totes les reserves del l'usuari amb id=id_usuari, indicant el nom de la pista
POST: Afegeix una reserva de l'usuari "id_usuari", els detalls venen a un JSON
DELETE: Elimina una reserva de l'usuari "id_usuari", els detalls venen a un JSON
